## コスト最適化戦略

### 1. 埋め込みコスト削減

```
現状: 全テキストを毎回 text-embedding-3-large で埋め込み
目標: 同一テキストの再埋め込みゼロ、次元数オプション化

対策:
├── SHA-256 キャッシュ: テキストハッシュ → 埋め込みベクトル
├── 次元選択: 3072 (高精度) / 1536 (バランス) / 256 (高速)
├── バッチ最適化: 重複テキスト除外してからAPI呼び出し
└── 差分埋め込み: 変更チャンクのみ再埋め込み

期待効果: API コスト 60-80% 削減
```

### 2. LLM コスト削減

```
段階的モデル選択:
├── Haiku  ($0.25/1M tokens): クエリ展開、分類、タグ付け
├── Sonnet ($3/1M tokens): エンティティ抽出、要約
├── Opus  ($15/1M tokens): メモリ統合、高品質推論
└── ローカル (Ollama): 機密データ処理、オフライン

ルーティング基準:
├── 入力長 < 500 tokens → Haiku
├── 構造化抽出タスク → Sonnet
├── 複雑な推論/統合 → Opus
└── APIキーなし → ローカルフォールバック

期待効果: LLM コスト 70-90% 削減
```

### 3. ストレージ最適化

```
├── 埋め込みベクトル量子化: float32 → int8 (4倍圧縮)
├── タイムライン圧縮: 90日以上前のエントリを圧縮アーカイブ
├── チャンクインデックス: 使用頻度の低いチャンクをディスクに退避
└── PGLite WAL 最適化: checkpoint interval調整

期待効果: ストレージ 50-70% 削減
```

### UX 原則

1. **3分ルール**: オンボーディング全体を3分以内に完了
2. **即時フィードバック**: 各操作で即座に結果を表示
3. **スキップ可能**: 全ステップにスキップオプション
4. **ローカル安心**: データがローカルにのみ保存されることを明示
5. **プログレス可視化**: 進捗バーで残りステップを表示
6. **エラー復帰**: 失敗時は具体的な解決策を提示
